## Design Context

### Users

- **1인 예비 회계사** (CPA 준비생 / 회계법인 입사 직전)
- 회계 개념(자산·부채·자본·수익·비용, 재무제표 3종, 현금흐름)을 **손에 쥐고 돌려보며** 체감하려는 학습자. 실무 시스템(더존·SAP)의 대체재가 아닌 **축소판 시뮬레이터**.
- 사용 맥락: 혼자, 공개 URL, 데스크톱이 주. 낮엔 학습/실습, 밤엔 복습. 데이터 민감도는 낮음(실제 기업 장부 아님).
- Jobs to be done:
  1. 가상 거래를 입력(또는 엑셀 일괄 업로드)하고
  2. 그것이 손익계산서·재무상태표·현금흐름표로 **어떻게 흘러가는지** 한눈에 확인
  3. 카테고리·계정과목 구조를 건드리며 개념 내면화

### Brand Personality

- **3 단어**: *차분하다 · 정돈되어 있다 · 신뢰가 간다*
- **Calm confidence** — 과장 없이 정확. 숫자를 검토할 때 방해받지 않고 집중할 수 있는 톤. "잘 정돈된 라이브러리·연구실" 느낌.
- 사용자는 이 앱을 쓰며 **"내가 진짜 회계를 다루고 있다"** 는 감각을 받아야 함. 게임처럼 가볍지도, 금융 앱처럼 공격적이지도 않음.

### Aesthetic Direction

- **Archival / Ledger** — 종이·잉크 질감의 디지털 장부. 19–20세기 회계 장부·도서관 카탈로그·관보 레이아웃의 정돈된 무게감을 재해석.
- **Anti-references** (절대 피할 것):
  - Crypto/AI SaaS 다크 + 네온 (검정 배경 + 시안/보라 글로우) — Brand Anti
  - QuickBooks·Zoho·삼일 SaaS형 카드 그리드 대시보드 — 지금 상태가 이쪽에 가까우니 이탈 필요
  - AI 기본 블루(#3b82f6/#2563eb), 이모지 아이콘, gradient text, 사이드 스트라이프 보더
- **Theme**: 라이트 + 다크 **둘 다**. 사용자 토글.
  - **Light**: 크림/워밍된 오프화이트 종이 위에 잉크색 타입. 순수 white(#fff) 금지.
  - **Dark**: 네온 다크 아님. "램프 아래 장부" — 짙은 잉크블루 또는 따뜻한 목탄색 배경 + 크림색 타입.
- **Palette 방향** (OKLCH, 뉴트럴은 브랜드 휴로 미세 틴트):
  - **Paper / Surface**: 워밍 크림 (light) · 잉크블루 차콜 (dark)
  - **Ink (기본 텍스트)**: 따뜻한 near-black (light) · 크림 (dark)
  - **Accent (브랜드)**: *oxblood/burgundy* 또는 *prussian blue* 중 하나 — 고전 장부 커버/잉크색. AI 블루 금지.
  - **Income (수입·증가)**: 회계사 "green eyeshade"를 반영한 **숲 녹색** (탁한 무광). 네온 emerald 금지.
  - **Expense (지출·감소)**: 잉크 레드 — 채도 낮은 적갈색. `#ef4444` 같은 원색 금지.
  - **Supporting**: 세피아·골드·머스터드 등 페이퍼 톤 악센트 1–2개 제한.
- **Typography 방향**:
  - **Display/Heading**: Serif. Pretendard·Inter·DM Sans 류 금지. Fraunces/Newsreader/Lora/Crimson/Cormorant/Playfair/IBM Plex 계열 모두 금지. Spectral, Vollkorn, Source Serif 4, Libre Caslon, RIDIBatang, Noto Serif KR 등 장부·문서 계열에서 선택.
  - **Body (UI·한글)**: 기존 Pretendard 유지 가능(가독성 이유). 단 h1/h2/h3·숫자 강조는 세리프로 교체해 위계 차별화.
  - **Numbers**: `font-variant-numeric: tabular-nums` 필수. 필요 시 모노스페이스 병용(JetBrains Mono, Monaspace Xenon, DM Mono 등 — IBM Plex Mono·Space Mono 금지).
- **Layout**: 카드 남발 중단. 현재 모든 섹션이 `.card`로 포장되어 있음 → 배경 틴트·얇은 1px 구분선·제목 위계로 위계 만들기. 테이블과 숫자가 주인공.

### Design Principles

1. **Numbers first, chrome last.** 장식은 숫자의 가독성을 해치지 않는 선까지만. 테이블 정렬·tabular numbers·우측 정렬 금액이 디자인의 중심.
2. **Quiet confidence over hero metrics.** "큰 숫자 3개" KPI 블록 패턴 탈피. 비교·추세·구성이 더 중요한 자리에서는 표나 리스트가 카드보다 강하다.
3. **Paper over panels.** 카드 중첩/그림자 남발 대신 종이 같은 연속 표면 + 타이포그래피 위계. 구분선 1px이 shadow-lg 하나보다 낫다.
4. **Warm neutrals, restrained accent.** 뉴트럴은 크림/차콜 워밍 계열. 브랜드 악센트는 10% 규칙 — 핵심 행위(조회/저장), 양수/음수 금액, 활성 탭 정도에만.
5. **Two themes, one soul.** 라이트·다크가 같은 톤을 공유해야 함. 다크는 라이트의 색 반전이 아니라 "밤에 본 같은 장부". 네온·글로우 금지.
6. **Accessibility is not decoration.** 라벨-인풋 연결, 키보드 포커스 링, 44px 터치 타겟, WCAG AA 대비 — 타협 없음.

### Accessibility Requirements

- WCAG 2.1 **Level AA** 준수 (텍스트 대비 4.5:1, UI 컴포넌트 3:1).
- 키보드 전용 탐색 가능. 모달은 `role="dialog"` + `aria-modal` + focus trap + ESC.
- `prefers-reduced-motion` 존중.
- 색만으로 상태를 표현하지 않기(수입/지출 배지는 텍스트 라벨 동반 — 현재 구현 유지).

### Out of Scope (by design)

- 로그인·다국어·다통화·부가세·감사추적 — 제품 범위상 없음. 디자인이 이것을 있는 척 꾸미면 안 됨.
