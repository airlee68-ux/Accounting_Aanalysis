### 백엔드 API

```
GET  /api/imports/transactions/template    → 엑셀 템플릿 다운로드 (샘플 3행 포함)
POST /api/imports/transactions/preview     → 업로드 파일 파싱 + 검증 (저장 X)
POST /api/imports/transactions/confirm     → 유효 행만 일괄 저장
```

**템플릿 컬럼:** `날짜 / 구분 / 내용 / 금액 / 카테고리 / 메모`

**검증 항목:**
- 날짜 형식 (`YYYY-MM-DD`, `YYYY/MM/DD`, `YYYY.MM.DD`, `YYYYMMDD` 허용)
- 구분 = `수입`/`지출` (또는 `INCOME`/`EXPENSE`)
- 내용 필수
- 금액 > 0 (쉼표·"원" 제거 후 파싱)
- 카테고리 이름 → id 자동 매칭 (없으면 에러 표시)

### 프론트엔드 UX
- 거래 목록 카드 우측 상단에 **[템플릿 다운로드]** + **[엑셀 업로드]** 버튼 배치
- 업로드 → 미리보기 모달: 전체/유효/오류 건수 + 행별 상태 표시 (오류 행은 빨간 배경)
- **[N건 저장]** 버튼으로 유효한 행만 커밋

## 3. seed.py 버그 수정

### 증상
프로덕션 거래 목록에 **4월 23일, 4월 24일**(오늘=4/20 이후의 미래) 거래 2건이 존재.

### 원인
[`seed.py`](../backend/app/seed.py)에서:
```python
base = today.replace(day=1) - timedelta(days=month_back * 30)
date=base + timedelta(days=random.randint(0, 27))
```
→ 이번 달(`month_back=0`)은 `base = 4/1`이고 offset 0~27이므로 **최대 4/28**까지 생성 가능.

### 수정
`sample_date()` 헬퍼를 도입, **이번 달에 한해** offset 최대치를 `today.day - base.day`로 제한. 과거 달은 기존 0~27 범위 유지.

**관련 커밋:** `fdf73e9` — "Fix seed generating future-dated transactions"

### 프로덕션 DB 정리
미래 날짜 레코드 2건은 사용자가 UI의 "삭제" 버튼으로 직접 제거 완료.
